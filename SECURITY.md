# Security Policy

## Supported versions

Security fixes are generally applied to the latest supported release line.

| Version | Supported |
|---|---|
| latest release | yes |
| older releases | best effort only |

## Reporting a vulnerability

Please do **not** open a public GitHub issue for security-sensitive problems.

Preferred reporting path:
- use GitHub private vulnerability reporting for this repository if it is enabled

Fallback:
- contact the maintainer privately through GitHub before opening any public report

## What to include

Please include:
- affected version
- device / Android version if relevant
- clear reproduction steps
- proof of concept if appropriate
- impact assessment if known

## Response expectations

Best effort:
- acknowledge receipt within a reasonable time
- reproduce and assess severity
- fix privately where appropriate
- publish a coordinated fix or release note when ready
