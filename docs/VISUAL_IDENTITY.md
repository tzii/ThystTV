# ThystTV Visual Identity

## Positioning

ThystTV should feel like a polished, open-source Android viewer app: dark, fast, clean, technical, and viewer-first.

Avoid copying Twitch branding. The app can reference streaming/player UX, but it should have its own visual identity.

## Core Look

- Dark Android app aesthetic
- Purple/violet gradients
- Cyan highlights
- Clean device frames
- Real screenshots preferred over fake UI
- GitHub/open-source friendly presentation
- Sharp, modern, not corporate SaaS bland
- Not gamer-cringe
- Not neon overload
- Not a Twitch clone

## Hero / Banner Direction

Good directions:

- polished phone/device mockup
- playback speed visible
- quality popup or speed popup as a feature callout
- floating translucent chat overlay
- local stats as a differentiator
- subtle purple/cyan lighting
- GitHub-friendly open-source feel
- no ugly black bars in showcased player states

Avoid:

- Twitch logo or Twitch wordmark
- random streamer faces unless from safe screenshots
- fake impossible UI
- messy collage
- unreadable tiny screenshots
- generic AI-app gradients
- excessive glow

## Screenshot Rules

- Prefer real screenshots.
- Wrap screenshots in clean device frames when useful.
- Keep feature popups readable.
- Do not crop away important UI state.
- Do not show private account data.
- Avoid notches unless the target device actually has one.
- Use screenshots that demonstrate real features:
  - playback speed
  - quality selection
  - floating chat
  - minimized player
  - stats
  - large-screen behavior

## Recent Visual Lessons

- Launcher icon backgrounds need enough contrast so the purple mark does not blend into the background.
- Player menus should follow one coherent Material direction: clean chips, theme-aware sheets, no awkward shadows or nested scrollbars.
- Updater/changelog content needs rendered Markdown, not raw Markdown text.
- Stats range controls must avoid clipped text and stray divider lines.
- Floating chat is a video overlay, so it should prioritize readability over matching the light app surface.

## README/Site Rules

- Screenshots should support the narrative.
- Do not duplicate the same navigation concepts in multiple places.
- Docs and Contribute should serve different purposes.
- Use concise captions.
- Prefer one polished hero over many mediocre images.

## Asset Safety

Never commit:

- private screenshots
- account tokens
- personal notifications
- unreleased private APK links
- third-party logos used in a misleading way
