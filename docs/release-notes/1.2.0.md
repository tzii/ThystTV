# ThystTV 1.2.0

## Highlights
- Player polish focused on smoother daily viewing, faster VoD navigation, and cleaner fullscreen controls.
- ThystTV-owned update and changelog flow, including release-note rendering and download progress.
- Refreshed GitHub Pages landing page and README with real app screenshots from the 1.2 asset package.
- Release workflow, maintainer docs, issue templates, and project documentation are now ready for tag-based releases.

## Player improvements
- The fullscreen playback speed control now shows the active speed.
- Fullscreen VoD scrubbing is duration-aware so long videos can be traversed much faster.
- The playback speed popup was tightened up for portrait and landscape use.
- The playback quality picker now uses a custom ThystTV dialog while preserving the existing quality-change behavior.
- Live quality labels are more readable, including utility modes such as audio-only and chat-only.
- Minimized player presentation was cleaned up to avoid broken-looking black bars on wide aspect ratios.

## Stats and layout
- Stats now include range filters for 7 days, 30 days, and all time.
- Stats range controls were polished and aligned with the app UI.
- Top-stream observer handling was tightened to avoid detached-view updates.

## Updates and release notes
- Update checks now point at ThystTV GitHub releases by default.
- Update detection uses release version information instead of relying only on asset timestamps.
- Update dialogs show release metadata, markdown-rendered notes, and clearer actions.
- APK download progress is surfaced during updater downloads.
- A dedicated changelog view provides bundled fallback notes when network release data is unavailable.

## Stability and upstream sync
- Media3/ExoPlayer was pinned back to the safer 1.9.3 line as a focused fix candidate for reported Android 15/Nubia player instability.
- The upstream Xtra sync remains selective. Broad player rewrites, Hilt removal, generated query churn, and repo-wide renames are deferred until after 1.2.
- The unraid message-id and message-type compatibility fixes from upstream are included or manually adapted.

## Project presentation
- README and GitHub Pages were refreshed with current ThystTV messaging and real screenshots.
- Community and maintainer files were added for contributions, security, governance, issues, and pull requests.
- CI and release workflows are split so public releases are tied to intentional version tags.

## Verification
- `assembleDebug` and `assembleRelease` are expected to pass before publishing.
- Manual QA should still cover live playback, VoDs, fullscreen scrubbing, speed and quality dialogs, floating chat, stats filters, updater/changelog flows, and README/site rendering.
