# ThystTV 1.2.0

ThystTV 1.2.0 is the first major milestone release for the fork. It moves the project from a working Xtra-based build into a more complete ThystTV release: a clearer player experience, an in-app updater and changelog flow, local stats polish, refreshed documentation, and a GitHub Pages front door for the project.

This release is also the first one shipped through ThystTV's own tag-based release workflow.

## Highlights
- New ThystTV updater flow with GitHub release checks, rendered release notes, download progress, and a dedicated changelog screen.
- Custom playback quality dialog with clearer video quality, audio-only, and chat-only controls.
- Fullscreen playback speed dialog and gesture feedback polish for daily viewing.
- Duration-aware VoD scrubbing so long videos can be navigated much faster.
- Local stats range filters for 7 days, 30 days, and all time.
- Refreshed README and GitHub Pages site with current screenshots, feature copy, and release-ready project docs.
- Release automation, issue templates, maintainer docs, security policy, governance notes, and CI/release workflow cleanup.

## Player and controls
- Added a custom ThystTV quality picker for fullscreen playback.
- Fixed the landscape audio-only/manual-quality edge case where the quality sheet could collapse to only `Auto` and utility modes.
- Filtered invalid numeric HLS quality fallback labels such as `0`.
- Improved live quality labels, including audio-only and chat-only modes.
- Tightened the playback speed popup for portrait and landscape layouts.
- Made fullscreen VoD scrubbing scale with video duration.
- Cleaned up minimized player presentation on wide aspect ratios.
- Preserved the 1.1.6 stream-switching and Media3 handoff fixes.

## Updater and changelog
- Update checks now point at ThystTV GitHub releases by default.
- Update detection uses release version information instead of relying only on APK asset timestamps.
- Update dialogs show release metadata, markdown-rendered notes, and clearer actions.
- APK download progress is shown during updater downloads.
- A dedicated changelog screen provides bundled fallback notes when network release data is unavailable.

## Stats and layout
- Added stats range filters for 7 days, 30 days, and all time.
- Polished stats controls and aligned them with the app UI.
- Tightened top-stream observer handling to avoid detached-view updates.
- Improved layout behavior for phones, tablets, and wider screens.

## Upstream Xtra sync
ThystTV 1.2.0 uses a selective upstream sync from [`crackededed/Xtra`](https://github.com/crackededed/Xtra). We intentionally avoided a broad upstream merge because several newer upstream changes overlap with ThystTV player, updater, and release-workflow work.

Included or patch-equivalent upstream commits:

- `e3bcad57` - Update strings.xml (#908)
- `9d630ce4` - fix stream download quality
- `579f87ac` - update proguard
- `582f58ef` - update unraid message id
- `06fd811b` - downgrade exoplayer
- `88bc97b4` - update unraid message type

Manually adapted from upstream:

- `628ba784` - show update download progress

Deferred for after 1.2:

- broad WIP player rewrites
- Hilt removal
- generated query/schema churn
- broad network refactors
- repo-wide rename churn

## Stability
- Media3/ExoPlayer was pinned back to the safer 1.9.3 line as a focused fix candidate for reported Android 15/Nubia player instability.
- The HLS parser compatibility rollback from upstream was kept narrow to reduce release risk.
- Chat compatibility now includes the upstream unraid message-id/message-type behavior.
